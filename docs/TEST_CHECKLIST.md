# Faceology 기능 테스트 체크리스트

- Updated: 2026-02-15
- 범위: iPhone(Expo) 클라이언트 + Node 서버 API
- 목적: MVP 기능 동작/오류 처리/정책 준수 확인

## 1) 사전 준비

- 서버 실행: `npm run dev:server`
- 클라이언트 실행: `npm run dev:client`
- iPhone 실기기 테스트 시:
  - iPhone과 서버 Mac을 같은 Wi-Fi에 연결
  - 앱 내 서버 주소를 `http://<Mac 로컬IP>:4000`으로 설정

## 2) 핵심 기능 테스트 (P0)

| ID | 구분 | 시나리오 | 절차 | 기대 결과 |
|---|---|---|---|---|
| F-001 | 앱 실행 | Expo 앱 정상 실행 | `npm run dev:client` 후 Expo Go로 접속 | 앱 첫 화면이 정상 표시됨 |
| F-002 | 서버 상태 | 헬스체크 응답 | `GET /api/health` 호출 | `status: ok` 응답 |
| F-003 | 동의 필수 | 동의 없이 분석 시도 | 동의 스위치 OFF 상태에서 분석 버튼 확인 | 분석 버튼 비활성화 |
| F-004 | 이미지 선택(카메라) | 카메라 촬영 선택 | `카메라 촬영` 버튼 탭 | 카메라 열림, 촬영 이미지 미리보기 표시 |
| F-005 | 이미지 선택(앨범) | 사진 앨범 선택 | `사진 앨범 선택` 버튼 탭 | 선택 이미지 미리보기 표시 |
| F-006 | 분석 성공 | 정상 이미지 분석 | 동의 ON + 이미지 선택 후 `분석 시작` | 결과 4개 카테고리 + 오늘의 한 줄 표시 |
| F-007 | variation 재생성 | 같은 사진 재생성 1~2회 | 분석 후 `같은 사진 재생성` 반복 | 최대 3회(0,1,2) 내 재생성, 이후 비활성화 |
| F-008 | 결과 공유 | 공유 시트 호출 | 결과 화면에서 `결과 공유` 탭 | iOS 공유 시트 표시 |
| F-009 | 결과 카드 열기 | SVG 카드 URL 오픈 | 결과 화면에서 `결과 카드 열기` 탭 | 브라우저에 카드 이미지 표시 |
| F-010 | 데이터 삭제 | 분석 데이터 삭제 | 결과 화면에서 `분석 데이터 삭제` 탭 | 삭제 성공 메시지 + 결과 상태 초기화 |
| F-011 | 삭제 후 카드 접근 | 삭제된 분석 카드 접근 | 삭제 후 기존 카드 URL 접속 | `ANALYSIS_NOT_FOUND`(404) |
| F-012 | 면책 문구 | 오락용 안내 노출 | 결과 화면 확인 | 면책 문구가 항상 표시됨 |

## 3) 입력 검증/오류 처리 테스트 (P0)

| ID | 구분 | 시나리오 | 절차 | 기대 결과 |
|---|---|---|---|---|
| V-001 | 동의 검증 | `consent=false` 요청 | `POST /api/analyze`에 `consent=false` | `CONSENT_REQUIRED`(400) |
| V-002 | 파일 누락 | image 없이 분석 요청 | `POST /api/analyze`에서 파일 제외 | `IMAGE_REQUIRED`(400) |
| V-003 | variation 범위 오류 | variation=3 요청 | `POST /api/analyze`에 3 전달 | `INVALID_VARIATION`(400) |
| V-004 | 확장자/타입 오류 | 비이미지 업로드 | txt 등 파일 업로드 | `UNSUPPORTED_FILE_TYPE`(400) |
| V-005 | 용량 초과 | 10MB 초과 이미지 업로드 | 10MB 초과 파일 업로드 | `FILE_TOO_LARGE`(400) |
| V-006 | 품질 실패 | 저품질 이미지 분석 | 저해상도/프레이밍 불량 이미지 업로드 | `QUALITY_CHECK_FAILED`(422) + 가이드 메시지 |
| V-007 | 삭제 요청 유효성 | analysisId 없이 삭제 | `POST /api/delete {}` | `ANALYSIS_ID_REQUIRED`(400) |
| V-008 | 없는 데이터 삭제 | 존재하지 않는 ID 삭제 | 임의 ID로 삭제 API 호출 | `ANALYSIS_NOT_FOUND`(404) |

## 4) 품질 체크 세부 테스트 (P1)

| ID | 구분 | 시나리오 | 절차 | 기대 결과 |
|---|---|---|---|---|
| Q-001 | 해상도 | 낮은 해상도 | 480x480 미만 이미지 업로드 | `low_resolution` 이슈 포함 |
| Q-002 | 밝기(어두움) | 매우 어두운 사진 | 조명 부족 사진 업로드 | `too_dark` 또는 `face_not_clear` |
| Q-003 | 밝기(과다) | 너무 밝은 사진 | 과노출 사진 업로드 | `too_bright` 또는 `face_not_clear` |
| Q-004 | 대비 | 대비가 낮은 사진 | 윤곽 흐린 사진 업로드 | `low_contrast` 이슈 포함 |
| Q-005 | 프레이밍 | 종횡비 부적절 | 지나치게 가로/세로 긴 이미지 | `poor_framing` 이슈 포함 |
| Q-006 | 다중 얼굴 의심 | 가로로 넓은 군상 사진 | 다중 인물 사진 업로드 | `multiple_faces_suspected` 이슈 포함 |

## 5) iPhone 실기기 테스트 (P0)

| ID | 구분 | 시나리오 | 절차 | 기대 결과 |
|---|---|---|---|---|
| I-001 | 네트워크 주소 | localhost 사용 시 실패 확인 | 실기기에서 API=`http://localhost:4000` | 통신 실패(예상 동작) |
| I-002 | 로컬IP 사용 | 로컬IP 변경 후 재시도 | API=`http://<Mac IP>:4000` 입력 | 분석 요청 정상 성공 |
| I-003 | 권한 거부(카메라) | 카메라 권한 거부 | 촬영 시 권한 거부 | 오류 메시지 표시, 앱 크래시 없음 |
| I-004 | 권한 거부(앨범) | 앨범 권한 거부 | 앨범 접근 시 권한 거부 | 오류 메시지 표시, 앱 크래시 없음 |

## 6) 회귀 테스트 최소 세트

- R-001: 동의 ON + 정상 이미지 분석 성공
- R-002: variation 2회 재생성 후 버튼 비활성화 확인
- R-003: 결과 공유 시트 호출 확인
- R-004: 결과 카드 열기 성공 확인
- R-005: 데이터 삭제 후 결과 상태 초기화 + 카드 404 확인
- R-006: 품질 실패 이미지에서 가이드 메시지 확인
